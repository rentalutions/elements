# Roadmap

We want to move away from the explicitly named 'rent_avail' packages and use a design system name that encompasses the work we do and is conceptually tied to real estate but doesn't reference a single product or audience.

Doors is one of those building blocks for customer rentals.

## Doors Design System

## Packages

| Package           | Description                                                                      |
| ----------------- | -------------------------------------------------------------------------------- |
| @doors/core       | Base providers and layout components.                                            |
| @doors/controls   | User interface controls and inputs.                                              |
| @doors/display    | Components used for data display.                                                |
| @doors/hooks      | Common react hooks.                                                              |
| @doors/icons      | Collection of branded svg icons.                                                 |
| @doors/overlays   | Components positioned outside of the normal document flow.                       |
| @doors/navigation | Components providing ways to break up a user process into more managable chunks. |
| @doors/utils      | Utility functions.                                                               |

## Patterns

| Pattern              | Description                                                                   |
| -------------------- | ----------------------------------------------------------------------------- |
| Address Autocomplete | Address search using the select component and the useAddressAutocomplete hook |
| Stepper              | Display content divided into a sequence.                                      |
